2018-06-19

原文地址：https://blog.csdn.net/nemo____/article/details/72897455

可解决大部分常规安装遇到的问题；

首先说明下 ：1） Centos7将默认数据库mysql替换成了Mariadb，如果想继续使用mysql 需要卸载Mariadb 再安装mysql

2） mysql 5.7版本的密码问题搞得人很头疼，所以干脆选择mysql5.6了

安装MySQL

  1. 下载安装包 mysql-5.6.36-linux-glibc2.5-x86_64.tar.gz

下载地址https://dev.mysql.com/downloads/mysql/5.6.html

选择如下选项

下载这个版本：

  1. 卸载系统自带的Mariadb

rpm -qa|grep mariadb //查询出已安装的mariadb

rpm -e –nodeps 文件名 //卸载,文件名为使用rpm -qa|grep mariadb 命令查出的所有文件

  1. 删除etc目录下的my.cnf文件

    rm /etc/my.cnf

  2. 执行以下命令来创建mysql用户组

groupadd mysql

  1. 执行以下命令来创建一个用户名为mysql的用户并加入mysql用户组

useradd -g mysql mysql

  1. 将下载的二进制压缩包放到/usr/local/目录下。

  2. 解压安装包

tar -zxvfmysql-5.6.36-linux-glibc2.5-x86_64.tar.gz

  1. 将解压好的文件夹重命名为mysql

到此数据库安装完毕！

配置MySQL

  1. 授予my.cnf的最大权限。

chown 777 /etc/my.cnf

设置开机自启动服务控制脚本：

  1. 复制启动脚本到资源目录

cp ./support-files/mysql.server /etc/rc.d/init.d/mysqld

  1. 增加mysqld服务控制脚本执行权限

chmod +x /etc/rc.d/init.d/mysqld

  1. 将mysqld服务加入到系统服务

chkconfig –add mysqld

  1. 检查mysqld服务是否已经生效

chkconfig –list mysqld

命令输出类似下面的结果：

mysqld 0:off 1:off 2:on 3:on 4:on 5:on 6:off

表明mysqld服务已经生效，在2、3、4、5运行级别随系统启动而自动启动，以后可以使用service命令控制mysql的启动和停止。

  1. 启动msql（停止mysqld服务：service mysqld stop）

service mysqld start

  1. 将mysql的bin目录加入PATH环境变量，编辑/etc/profile文件

vi /etc/profile

在文件最后添加如下信息：

export PATH=$PATH:/usr/local/mysql/bin

执行下面的命令使所做的更改生效：

. /etc/profile

  1. 以root账户登陆mysql，默认是没有密码

mysql -u root -p

  1. 设置root账户密码 注意下面的you password改成你的要修改的密码

    use mysql

update user set password=password(‘you password’) where user=’root’and host=’localhost’;

  1. 设置远程主机登录，注意下面的your username 和 your password改成你需要设置的用户和密码

GRANT ALL PRIVILEGES ON . TO’your username‘@’%’ IDENTIFIED BY ‘your password’ WITH GRANT OPTION;

FLUSH PRIVILEGES ;
